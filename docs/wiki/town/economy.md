# 经济

经济模型的目标是让资源、劳动、工资和行业现金流互相闭合。NPC 商人不应被理解成凭空刷货的自动售货机。

## 设计流水线

```text
NPC 需求
-> 食物篮子
-> 配方产出计划
-> 原料需求
-> 饲料和工具折旧
-> 农田/矿山/采集/设施供给
-> 工时和体力校验
-> 工资表
-> 行业现金流
-> group 财富捕获
```

## 核心假设

| 参数 | 当前值 |
|---|---:|
| 每人每日饱食需求 | 75 hunger |
| 工作站每日有人值守时间 | 8 游戏小时 |
| 每名工人每日可用体力 | 180 stamina |
| 劳动价格锚 | 2.5 银 / 游戏小时 |
| 农田利用率 | 65% |
| 农田照料产量系数 | 80% |
| 留种/复种预留 | 作物需求的 10% |

## 食物篮子

| 食物组 | 目标占比 | 默认食物 | 用途 |
|---|---:|---|---|
| 家庭主食 | 42% | 面粉 | 多数住户买面粉回家做饭 |
| 成品面包 | 10% | 面包 | 值守、旅人、采办和临时用餐 |
| 家庭肉食 | 12% | 生肉 | 住户买生肉回家烹饪 |
| 蛋类 | 6% | 蛋 | 补充蛋白 |
| 蔬果 | 20% | 番茄 | 农田多样化和短保质食物去处 |
| 保藏食品 | 10% | 盐渍菜汤 | 盐、燃料和酒馆加工需求 |

## 钱如何流动

行业现金流大致为：

```text
retained = revenue - materials - fixed_wages - owner_labor_draw - spoilage - tax_or_rent
capture = max(retained, 0) + fixed_wages + owner_labor_draw
```

不要把行业收入直接理解成个人财富。工资、老板劳动报酬、留存利润、地租和税都要分开看。
