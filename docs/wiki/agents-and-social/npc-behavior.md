# NPC 行为

NPC 是小镇模拟的核心。设计目标是 NPC 和玩家使用同一套种植、制造、交易和资源规则。

## 当前可见定位

当前版本中，NPC 主要承担：

- 小镇生活体。
- 职业、地点和行业链的承载者。
- 可被感知、可对话、可被后台上下文引用的角色。
- agent runtime 的执行对象。

## 双轨大脑

设计上每个 NPC 同时跑两类 session：

| 轨道 | 作用 |
|---|---|
| 反应轨 | 低延迟，把事件转成世界行动 |
| 策略轨 | 定期总结事件、计划、更新 working memory |

反应轨读取策略轨留下的简报，用较低成本持续推进当前计划。

## 当前限制

- 不要把 NPC 理解成完整任务树 NPC。
- 战斗中 LLM 不会逐帧决策，战斗系统规划为传统行为树。
- NPC 制造应走同一反应表，而不是 LLM 现场创造规则。
